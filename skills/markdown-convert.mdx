The Markdown Convert skill converts a markdown file to `docx`, `pdf`, or `html` using [pandoc](https://pandoc.org/). It infers the target format from the user's request, resolves the output path, runs the appropriate pandoc command, and confirms the result. YAML frontmatter in the source file (`title`, `author`, `date`) is handled natively by pandoc — no special configuration required.

## When the skill triggers

This skill activates whenever the user wants to export, convert, or transform a markdown file — including phrasing such as:

- "make a Word doc from this"
- "export to PDF"
- "save as HTML"
- "convert to docx"
- "I need to share this as a PDF"

## Workflow

<Steps>
  <Step title="Identify the input file">
    The user provides a path. If it's relative, resolve it from their working directory. If the path is ambiguous, ask for clarification.
  </Step>
  <Step title="Determine the output format">
    Infer from the user's request:

    - "Word" / "docx" / "document" → `docx`
    - "PDF" / "pdf" → `pdf`
    - "HTML" / "web" / "webpage" → `html`

    If the format is unclear, ask: *"Which format — docx, pdf, or html?"*
  </Step>
  <Step title="Determine the output path">
    Default to the same directory as the input file with the appropriate extension. Use the user's specified path or name if provided.
  </Step>
  <Step title="Run pandoc">
    Execute the conversion command using the Bash tool (see commands below).
  </Step>
  <Step title="Confirm success">
    Report the output file path to the user.
  </Step>
</Steps>

## Pandoc commands

<Tabs>
  <Tab title="docx">
    ```bash
    pandoc "<input.md>" -o "<output.docx>"
    ```
  </Tab>
  <Tab title="html">
    ```bash
    pandoc "<input.md>" -o "<output.html>" --standalone
    ```
    The `--standalone` flag produces a complete HTML document with `<html>`, `<head>`, and `<body>` tags.
  </Tab>
  <Tab title="pdf">
    PDF requires a PDF engine. Check which engines are available first:

    ```bash
    which xelatex pdflatex weasyprint 2>/dev/null
    ```

    Then try engines in this order until one succeeds:

    ```bash
    # Try 1: xelatex (most common with TeX distributions)
    pandoc "<input.md>" -o "<output.pdf>" --pdf-engine=xelatex

    # Try 2: pdflatex
    pandoc "<input.md>" -o "<output.pdf>" --pdf-engine=pdflatex

    # Try 3: weasyprint
    pandoc "<input.md>" -o "<output.pdf>" --pdf-engine=weasyprint
    ```
  </Tab>
</Tabs>

### PDF engine fallback order

If no PDF engine is found, tell the user:

> PDF conversion requires a LaTeX distribution (e.g. MacTeX, TinyTeX) or weasyprint. For the quickest install: `brew install --cask mactex-no-gui` or `pip install weasyprint`.

## YAML frontmatter support

Pandoc reads YAML frontmatter automatically. The `title`, `author`, and `date` fields become document metadata:

- **docx / pdf** — appear on a title page
- **html** — populate the `<title>` tag

No special handling is needed — pandoc handles this natively.

```markdown
---
title: "Quarterly Report"
author: "Jane Smith"
date: "2026-04-19"
---

## Introduction
...
```

## Notes and edge cases

<Note>
Always quote file paths in the pandoc command to handle spaces and special characters correctly.
</Note>

- If pandoc returns an error, show the full error message to the user and suggest a fix (missing engine, unsupported syntax, etc.)
- If the user asks to convert multiple files, handle them sequentially and confirm each one before moving to the next

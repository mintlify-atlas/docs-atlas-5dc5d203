---
title: "Quickstart: install your first Agent Skill in minutes"
description: "Learn how to install the bash skill for Claude Code or Kiro in under five minutes, and verify it works by asking your agent to write a shell script."
---

Agent Skills works by symlinking skill directories into your agent's skills folder. Once installed, the agent discovers and applies skills automatically when they're relevant — no extra prompting required. This guide walks you through cloning the repository and installing your first skill.

<Steps>
  <Step title="Clone the repository">
    Clone the `agent-skills` repository once to a local directory. You'll install individual skills from here using symlinks.

    ```bash
    git clone git@github.com:dmorand17/agent-skills.git
    cd agent-skills
    ```
  </Step>
  <Step title="Symlink a skill into your agent's skills folder">
    Create a symlink from the skill directory into `~/.claude/skills/` (for Claude Code) or `~/.kiro/skills/` (for Kiro). The example below installs the `bash` skill globally.

    <Tabs>
      <Tab title="Claude Code">
        ```bash
        ln -s $(pwd)/bash ~/.claude/skills/bash
        ```
      </Tab>
      <Tab title="Kiro">
        ```bash
        ln -s $(pwd)/bash ~/.kiro/skills/bash
        ```
      </Tab>
    </Tabs>
  </Step>
  <Step title="Verify the symlink is in place">
    Confirm the skill directory is visible in your agent's skills folder:

    ```bash
    ls ~/.claude/skills/
    # bash -> /path/to/agent-skills/bash
    ```
  </Step>
  <Step title="Try it out">
    Open Claude Code (or Kiro) and ask your agent to write a bash script:

    > "Write a bash script that backs up a directory to a timestamped archive."

    The agent should automatically apply the bash skill's conventions: `#!/usr/bin/env bash`, `set -euo pipefail`, a `main()` entry point, usage function, argument parsing, dependency checking, and timestamped `log_*` logging.
  </Step>
</Steps>

## Install all skills at once

To install every skill in a single command, use this loop:

<CodeGroup>
```bash Claude Code
for skill in bash skill-creator git-commit markdown-convert project-memory terraform; do
  ln -sf "$(pwd)/$skill" ~/.claude/skills/$skill
done
```

```bash Kiro
for skill in bash skill-creator git-commit markdown-convert project-memory terraform; do
  ln -sf "$(pwd)/$skill" ~/.kiro/skills/$skill
done
```
</CodeGroup>

## Project-scoped installation

Install a skill for a single project instead of globally by symlinking into the project's local skills directory:

<Tabs>
  <Tab title="Claude Code">
    ```bash
    ln -s $(pwd)/terraform /path/to/project/.claude/skills/terraform
    ```
  </Tab>
  <Tab title="Kiro">
    ```bash
    ln -s $(pwd)/terraform /path/to/project/.kiro/skills/terraform
    ```
  </Tab>
</Tabs>

## Keeping skills up to date

Because you installed skills via symlinks, updating is a single `git pull`:

```bash
cd agent-skills && git pull
```

All installed skills update automatically — no re-linking needed.
